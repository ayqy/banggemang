# "找回"源码计划 - 需求调查文档

## 一、项目背景

腾讯 QQ 浏览器"帮小忙"在线工具箱平台 (https://tool.browser.qq.com/) 是一个包含数百个在线工具的聚合平台。需要完整复刻该网站的所有功能和页面。

## 二、网站结构分析

### 2.1 整体架构

```
网站采用单页应用 (SPA) 架构
- 统一的左侧导航栏 (left-nav)
- 统一的顶部搜索区域
- 统一的右侧快捷导航 (回顶部、QQ 群、共建、反馈)
- 统一的页脚 (footer)
- 主要内容区域根据路由动态加载
```

### 2.2 页面类型

1. **首页/全部工具页** (`/`)
   - 展示所有工具的分类网格
   - 包含"最新工具"和"最热工具"推荐区域

2. **分类页面** (`/category/{categoryName}`)
   - 展示特定分类下的所有工具
   - 共 10 个分类：
     - 全部 (`/`)
     - 图片工具 (`/category/img`)
     - PDF 转换工具 (`/category/pdf`)
     - 数据换算工具 (`/category/data`)
     - 生活娱乐工具 (`/category/life`)
     - 教育工具 (`/category/education`)
     - 文本工具 (`/category/text`)
     - 文档转换工具 (`/category/doc`)
     - 开发工具 (`/category/develop`)
     - 视频工具 (`/category/video`)
     - 浏览器插件 (`/category/pc_plugin`)

3. **工具详情页面** (`/{toolName}.html`)
   - 每个工具一个独立页面
   - 包含工具介绍、使用方法、操作区域
   - 底部有"更多推荐"区域

## 三、UI 布局分析

### 3.1 整体布局

```
+------------------+------------------------------------------+
|                  |                                          |
|   左侧导航栏     |            顶部搜索区域                   |
|   (固定宽度)     |                                          |
|                  +------------------------------------------+
|   - Logo         |                                          |
|   - 分类菜单     |            主要内容区域                   |
|                  |                                          |
|                  |                                          |
|                  |                                          |
|                  +------------------------------------------+
|                  |                                          |
|                  |              页脚                        |
+------------------+------------------------------------------+
```

### 3.2 左侧导航栏

- Logo 区域：包含图标和"帮小忙 腾讯 QQ 浏览器在线工具箱"文字
- 分类菜单：10 个分类，每个分类有对应的 icon
- 当前选中分类高亮显示（淡紫色背景）

### 3.3 顶部区域

- 搜索框：带搜索图标，支持工具搜索
- 统计信息：显示"工具箱已累计帮助了 XXX 人次"
- 分享按钮

### 3.4 右侧快捷导航

- 回顶部按钮
- QQ 群链接
- 共建链接
- 反馈链接

### 3.5 工具卡片（分类页）

每个工具卡片包含：
- 工具 icon（圆形，带背景色）
- 工具名称（标题）
- 工具描述（简短介绍，最多 2 行）
- 标签（可选）：new、hot、推荐、权益卡、限免等

### 3.6 工具详情页结构

```
+----------------------------------------------------------+
|  工具 icon + 工具名称                                      |
+----------------------------------------------------------+
|                                                          |
|                    工具操作区域                           |
|                    (每个工具不同)                          |
|                                                          |
+----------------------------------------------------------+
|  工具介绍及使用方法                                        |
|  - 工具描述                                               |
|  - 使用步骤                                               |
+----------------------------------------------------------+
|  更多推荐                                                 |
|  - 推荐工具 1                                            |
|  - 推荐工具 2                                            |
+----------------------------------------------------------+
|  页脚                                                    |
+----------------------------------------------------------+
```

## 四、样式分析

### 4.1 颜色方案

- 主色调：蓝色 (#4080FF)
- 背景色：
  - 左侧导航背景：#F5F6FA
  - 页面背景：#FFFFFF
  - 卡片背景：#FFFFFF
- 文字颜色：
  - 主标题：#1D1D1F
  - 正文：#86909C
  - 链接：#4080FF
- 高亮色：淡紫色 (#E8E8FF) - 用于选中状态

### 4.2 字体

- 字体族：系统默认字体
- 字号：
  - 大标题：18-20px
  - 小标题：14-16px
  - 正文：12-14px

### 4.3 间距

- 卡片间距：16px
- 内边距：12-24px
- 圆角：8px

### 4.4 阴影

- 卡片阴影：0 2px 8px rgba(0, 0, 0, 0.08)
- 悬浮阴影：0 4px 16px rgba(0, 0, 0, 0.12)

## 七、功能需求

### 7.1 通用功能

1. **左侧导航**
   - 点击分类跳转到对应分类页面
   - 当前分类高亮显示

2. **搜索功能**
   - 输入关键词搜索工具
   - 实时显示搜索结果

3. **右侧快捷导航**
   - 回顶部：点击平滑滚动到页面顶部
   - QQ 群：点击打开 QQ 群链接
   - 共建：点击打开共建反馈页面
   - 反馈：点击打开反馈页面

4. **响应式布局**
   - 支持不同屏幕尺寸

### 7.2 分类页面功能

- 展示该分类下所有工具
- 工具卡片点击跳转到对应工具页面
- 工具卡片悬浮效果

### 7.3 工具页面功能

- 每个工具的具体功能实现
- 工具介绍和使用说明展示
- 更多推荐区域

## 八、技术选型建议

### 8.1 前端框架

考虑到要完整复刻原网站功能，建议采用：
- HTML5 + CSS3 + 原生 JavaScript
- 或者使用 Vue.js/React 等现代框架

### 8.2 样式方案

- 使用 CSS 变量管理颜色
- Flexbox/Grid布局
- 响应式设计

### 8.3 资源管理

- 下载所有图片资源到本地
- 整理 CSS 样式文件
- 模块化管理 JS 代码

## 九、开发计划

### 阶段一：基础架构
1. 创建项目目录结构
2. 实现通用布局组件（左侧导航、顶部搜索、页脚）
3. 实现 CSS 样式系统

### 阶段二：分类页面
1. 实现全部工具页面（首页）
2. 实现 10 个分类页面

### 阶段三：工具页面
1. 实现教育工具分类下的 16 个工具页面
2. 逐步实现其他工具页面

### 阶段四：测试优化
1. 功能测试
2. 样式优化
3. 性能优化

## 十、注意事项

1. 所有图片资源需要下载到本地
2. CSS 样式需要仔细分析和还原
3. 每个工具的具体功能需要单独分析实现
4. 保持与原网站完全一致的视觉效果和交互体验
